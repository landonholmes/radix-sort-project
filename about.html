About The Project:

This project is Radix Sort coded using JavaScript and HTML/CSS. It makes use of the JQuery and Underscore.js JavaScript libraries, concepts such as local storage (way better than cookies) and the window.performance system for accurate algorithm timing. It is not currently supported for IE but has been tested in the latest versions of Chrome and Firefox. IE hates localStorage and also does not support the standard function.name property.

The system defaults to certain input parameters upon first entry and generates an input based upon those parameters and runs radix sort on that input. The log on the right of the page displays what is happening in the JavaScript with status updates on what the input looks like before/after sorting and the time it took to sort for a specific set of parameters.

The user is given the option to change these parameters and re-run the sort using the new parameters along with a few controls on what is currently displayed in the log window. The reset button will reset the inputs back to the defaults of 1000, 1000, and 100. Hitting enter inside of an input will run the sort with the current input values. There is a checkbox to control whether or not the arrays being sorted are displayed before and after sorting. It is helpful to disable this display if the user has already seen that the sorting works on the input.

The radio buttons control how the input array is generated. The default is that the input is random. The user is able to choose if the input should be ordered or ordered backwards by default.

There are also checkboxes that enable the user to choose different comparison sorts to also run on the given input. If any of these options are chosen, they will display the sorting being done and the log will include an extra section showing an ordered list of the sorting runtimes. This allows the user to compare against a few other sorts to see what input runs best with which sort.

If more than one sort is chosen to compare against, below the log there will be a bar graph showing a visual representation of the run times for each sort.
